Synchronous
    Definition: Tasks are executed one after another, in sequence.

    Explanation:

    Each task must finish before the next one starts.

    If one task takes time (like reading a file), everything else waits.

    Example: In JavaScript, a normal function call is synchronous — the program pauses until it completes.

Asynchronous
    Definition: Tasks are executed independently, without blocking the main flow.

    Explanation:

    A task can start and then “hand off” its work (like I/O or network request) while the program continues running other tasks.

    When the task finishes, a callback, promise, or event loop handles the result.

    Example: In JavaScript, setTimeout() or fetch() are asynchronous — they don’t block the program while waiting.
